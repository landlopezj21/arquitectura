## Clasificación de sistemas

Antes de aplicar controles, cada sistema debe clasificarse:

| Nivel | Descripción | Ejemplos |
|---|---|---|
| **Crítico** | Datos personales sensibles, pagos, autenticación | Servicio de pagos, IdP, HR |
| **Interno** | Datos internos de la empresa sin exposición externa | Herramientas internas, dashboards |
| **Público** | Sin datos sensibles, acceso abierto | Sitio de marketing, documentación pública |

Los controles obligatorios varían según el nivel. Consulta la página de [controles](controles.md) para el detalle por nivel.

## Reporte de vulnerabilidades

Si descubres una vulnerabilidad de seguridad, reporta al canal privado de SOAINT.
